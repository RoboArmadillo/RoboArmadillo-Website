Documentation for the kit

Welcome to the Documentation of the RoPiPyKokiDuino kit. If you have made it this far I will assume that you have successfully managed to build the kit and install of the necessary dependancies on it. Congratulations! If not please head to the Download and Setup tab. Now for the fun bit we can start to add code to your robot.

To Start.

There are few things which we need to sort out first. From the folder, which you downloaded from Githhub during the setup instructions, take out the folder named "RoboArmadillo" and place that on a memory stick. The code is stored in the file "usercode.py" when you edit this it changes the code that your robot will run. At the beginning of this file you will at the very minimum need the following two lines of code

import robot

Don't worry too much about this, essentially its just importing the code which needs to run behind the scenes while you do your code which the robot executes.

R=robot.Robot()

This line creates what is called an "instance" of the robot object. This is just a container which has all of the functions which you will use to control your robot, in it.
